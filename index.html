WebDriver API

Chuẩn W3C — Viết lệnh bằng bất kỳ ngôn ngữ nào.

  1. 1. Viết lệnh

    Gọi driver.click() từ script

  2. 2. Đóng gói lệnh

    Chuyển thành HTTP POST gửi tới Server

  3. 3. Server diện giải

    Nhận lệnh, chọn Driver phù hợp, ra lệnh cho thiết bị

  4. 4. Thực thi hành động

    Thiết bị nhấp vào đúng Element được yêu cầu

Mô-đun hóa Hoàn Toàn Từ 2.x

MỚI

Tách riêng Server, Driver, Plugin — cài đặt độc lập.

Server (Manager)

Tiếp nhận REST API, điều phối luồng.

Driver (Translator)

Chuyển lệnh WebDriver → lệnh native.

Agent (Executor)

Thực thi trên thiết bị, khởi động app.

Điểm khác biệt với 1.x

  • • Cài Driver riêng: Chỉ cài những gì cần dùng.
  • • Cập nhật độc lập: Không chờ Server mới.
  • • Mở rộng Plugin: Tự viết Plugin tùy chỉnh.

Luồng Hoạt Động: Từ Code Đến Kết Quả

  1. 1. Khởi động Server

    Appium lắng nghe tại cổng mặc định 4723.

  2. 2. Gửi Desired Capabilities

    Client gửi JSON cấu hình: "Test app X trên Android Y".

  3. 3. Khởi tạo Session

    Server tạo Session ID, khởi động thiết bị/giả lập.

  4. 4. Thực thi lệnh

    Click, Swipe được gửi liên tục qua Driver/Agent.

  5. 5. Báo cáo

    Kết quả (Pass/Fail) gửi ngược về Client → Report.

Ưu & Nhược Điểm

Ưu Điểm

  • ●Code 1 lần, chạy nhiều nơi (cross-platform).
  • ●Tự do chọn ngôn ngữ (Java, Python, JS, C#...).
  • ●Cộng đồng hỗ trợ cực lớn, mã nguồn mở miễn phí.

Nhược Điểm

  • ●Tốc độ chậm hơn Native thuần.
  • ●Cài đặt môi trường phức tạp.
  • ●Đôi khi không ổn định (flaky tests).

Appium Inspector — Công cụ "phải có"

Giao diện đồ họa để soi cấu trúc UI của app, tìm các mã định danh (ID, Xpath, Accessibility ID) nhằm viết code chính xác.

Các Bước Cài Đặt (Terminal)

# 1. Cài đặt Server

$ npm install -g appium

# 2. Cài đặt Driver (Android)

$ appium driver install uiautomator2

# 3. Khởi chạy Server

$ appium

Lưu ý: Thiết lập biến môi trường JAVA_HOME và ANDROID_HOME!
